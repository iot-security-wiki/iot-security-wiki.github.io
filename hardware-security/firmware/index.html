2.3 固件分析

固件分析

固件( Firmware )就是写入 EROM（可擦写只读存储器）或 EEPROM (电可擦可编程只读存储器)中的程序，修改固件是在提取出固件之后，一般是 16 进制的 BIN 文件或者 HEX 文件，然后通过逆向分析或解包之后，获得实际的逻辑代码，然后根据需求修改其中的关键代码，之后再重打包刷回芯片中，使硬件执行修改之后的固件。
